  • Programming Assignments
    1. WebSocketFrame Serialization (Due 30 Jan) - Individually implement the WebSocketFrame class with (de)serialization. Individually implement JUnit testing for your WebSocketFrame class.
      Team Left: Determine interface (24 Jan)
      Team Right: Determine protocol functionality to initially include (23 Jan)
    2. Ultimate JUnit Test (Due 4 Feb) - Team Left and Team Right implement the ultimate JUnit test for the WebSocketFrame class. Tests should:
      • include happy path and edge case serialization and deserialization
      • demonstrate proper JUnit 4 design (e.g., exception handling, parameterized tests, independent, single focus, simple, etc.)
      • provide full coverage of the public interface
      • be well organized, properly/consistently named, commented, and JavaDoc'd
      • include a test suite if you have multiple unit test classes
    3. WebSocket Server (Due 18 Feb) - Individually implement a WebSocket server that utilizes your model.WebSocketFrame serialization/deserialization. The server should accept connections and facilitate sending/receiving from/to a web socket server application. Your WebSocket server should expose an API for server applications. It should allow multiple applications to run on the server simultaneously. To demonstrate your application hosting ability, run a multiclient chat (WSMC) server application. Clients (browser) will send messages to the WSMC server application, which will repeat the message to all chat clients.

      Your WebSocket Server should be implemented in a class called server.WebSocketServer. This class should implement the Runnable interface where the run() method starts the dispatch loop of the server. This class must also implement a main() method that creates a thread that executes run().

      Team Left: Update and publish the WebSocketFrame interface as necessary. Write a web socket server application using the WebSocketApp interface. Provide this server implementation to Team Right by no later than 13 Feb.
      Team Right: Create and publish the application.WebSocketApp interface for web socket applications running on a web socket server (see the WebSocket API). Provide this interface specification to Team Left by no later than 10 Feb. Also, implement a usable web page to serve as the WSMC client application. Make sure all works on Google's Chrome browser. Provide this client implementation to Team Left by no later than 13 Feb.

    4. WebSocket Client/Tester (Due 4 Mar) - Individually implement a WebSocket client that implements the multichat client on the command line. The client should be contained in the "client" package with a main class named "WebSocketClientMulticlientChat". Your (de)serialization code must pass all of my tests.

      Teams Left and Right must implement the following:

      • JUnit-based WebSocket Server Correctness Test. You may not use your serialization/deserialization code for this.
      • WebSocket Server Scale Test. Each group must use a test framework (e.g., jmeter). Each group must use a different test framework (FCFS). You must provide both developer and user documentation for your test framework.
      Individually provide an analysis of your previously-submitted WebSocket server.
    5. Scalable WebSocket Server (Due 25 Mar) - Individually implement a non-blocking I/O version of your WebSocket server. Your server should add a new application called SlowMo that receives an echo request, sleeps for 2 seconds, and replies by echoing the message. Your Multi-client chat should continue to operate. Use your group's scale testing framework to test the scalability of your application. Include a report about your server's scalability, comparing your old and new version.
